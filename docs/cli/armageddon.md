# armageddon

The `armageddon` command generates Arma configuration material and supports orderer test/load workflows.

## armageddon
```
usage: armageddon [<flags>] <command> [<args> ...]

Utility for generating Arma config material

Flags:
  --help  Show context-sensitive help (also try --help-long and --help-man).

Commands:
  help [<command>...]
    Show help.

  generate [<flags>]
    Generate config material

  showtemplate
    Show the default configuration template needed to build Arma config material

  version
    Show version information

  submit [<flags>]
    Submit txs to routers and verify the submission

  load [<flags>]
    Submit txs to routers and verify the routers have received the txs

  receive [<flags>]
    Pull txs from some assembler and report statistics

  createSharedConfigProto [<flags>]
    Create a shared config binary from a shared configuration YAML file

  createBlock [<flags>]
    Create a new block with the given shared configuration
```


## armageddon generate
```
usage: armageddon generate [<flags>]

Generate config material

Flags:
  --help                  Show context-sensitive help (also try --help-long and
                          --help-man).
  --output="arma-config"  The output directory in which to place config files
  --config=CONFIG         The configuration template to use
  --useTLS                Defines if the connection between a client to a router
                          and an assembler is a TLS one or not
  --sampleConfigPath=SAMPLECONFIGPATH
                          The path to the sample config files
  --clientSignatureVerificationRequired
                          Specify if client signature verification is required
```
